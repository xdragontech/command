This guide covers the current feature set in admin-web, including first-run setup, content operations, and phase-1 scheduling.

1. Getting Started

What It Is

Command is the administrative backoffice for managing brands, operators, public client accounts, content, leads, diagnostics, and schedules.

What It Does

  • Controls who can access the backoffice
  • Controls brand runtime settings such as hostnames and brand email configuration
  • Stores and manages prompts, guides, and articles
  • Shows lead, analytics, and scheduling data

How To Use It

  1. On a new install, complete /setup first.
  2. Open the backoffice sign-in page.
  3. Sign in with your backoffice credentials.
  4. If MFA is enabled on your account, complete the authenticator code challenge.
  5. Use the left navigation to move between Dashboard, Accounts, Library, Scheduling, and Settings.
Superadmins can manage all brands and operator accounts. Staff users are limited to their assigned brands.

2. First-Run Setup

What It Is

The guided initialization flow for a new install.

What It Does

  • Creates the install profile
  • Creates the primary brand and its host mapping
  • Stores brand email metadata
  • Creates the protected bootstrap superadmin record

How To Use It

  1. Open /setup on an uninitialized install.
  2. Unlock setup using BACKOFFICE_BOOTSTRAP_PASSWORD.
  3. Complete the install name, brand, host, and brand email details.
  4. Review the completion handoff values before continuing to sign-in.

3. Dashboard

What It Is

The operational summary screen for signups, logins, tracked login IPs, leads, and chat leads.

What It Does

  • Shows period-based metrics for Today, 7 Days, and 30 Days
  • Shows trends over the selected period
  • Shows top signup countries and tracked login IPs

How To Use It

  1. Open Dashboard.
  2. Select the time period.
  3. Review the headline metrics, trend chart, country summary, and IP activity table.
  4. Use this page for monitoring, not editing.

4. Analytics

What It Is

A read-only lead analytics view built from the lead-event stream.

What It Does

  • Shows deduped total leads
  • Separates contact leads and chat leads
  • Shows period summaries and brand breakdowns

How To Use It

  1. Open Analytics.
  2. Refresh the view when you need the latest totals.
  3. Use it to understand lead volume and source mix, not to edit lead data.

5. Leads

What It Is

A searchable operational table of recent contact and chat leads.

What It Does

  • Filters by source and brand
  • Searches by name, email, and displayed values
  • Exports the visible result set to CSV or JSON
  • Lets operators inspect the raw payload for a lead row

How To Use It

  1. Open Leads.
  2. Filter and search as needed.
  3. Inspect the row details or export the current result set.

6. Staff Accounts

What It Is

The operator-management screen for staff and superadmin identities.

What It Does

  • Creates staff and superadmin accounts
  • Assigns brand access to staff
  • Blocks, unblocks, and deletes accounts
  • Generates invite links and password-reset links
  • Resets authenticator MFA when needed

How To Use It

  1. Open Accounts / Staff.
  2. Select an existing user or click Add Staff.
  3. Set role, email, username, password, and brand assignments.
  4. Use Create & Invite for invitation-based onboarding.

7. Client Accounts

What It Is

The management screen for brand-scoped external/public users.

What It Does

  • Creates and edits external client accounts
  • Assigns them to a brand
  • Marks email as verified when needed
  • Blocks, unblocks, or deletes accounts

How To Use It

  1. Open Accounts / Clients.
  2. Select a client or click Add Client.
  3. Set name, email, brand, and sign-in status.
  4. Use Mark Verified only when you intend to bypass the normal verification flow.

8. Brands

What It Is

The live source-of-truth screen for brand identity, runtime host mapping, and brand email configuration.

What It Does

  • Creates and edits brands
  • Stores production and preview public/admin hosts
  • Stores apex routing
  • Stores brand email settings such as status, sender identity, support email, and provider secret reference

How To Use It

  1. Open Settings / Brands.
  2. Select a brand or click Add Brand.
  3. Configure identity, hosts, and brand email metadata.
  4. Save carefully. Host and email mistakes affect live runtime behavior.

9. Configs

What It Is

A read-only diagnostics screen for install-level runtime and service configuration.

What It Does

  • Shows env presence or absence
  • Shows masked or summarized secret state
  • Shows runtime status and database connectivity checks
  • Shows what the running installation is actually using

How To Use It

  1. Open Settings / Configs.
  2. Review runtime, auth, setup, and service groups.
  3. Use this screen to verify state, not to edit values.

10. Security

What It Is

The per-operator MFA management screen for authenticator-app enrollment.

What It Does

  • Shows MFA status and enrollment state
  • Starts MFA setup
  • Displays QR code, setup secret, setup URI, and recovery codes during enrollment
  • Verifies setup with a 6-digit authenticator code

How To Use It

  1. Open Settings / Security.
  2. Start setup if MFA is disabled.
  3. Scan the QR code and save the recovery codes.
  4. Enter a valid 6-digit authenticator code to finish enrollment.

11. Prompt Library

What It Is

The brand-scoped prompt management area.

What It Does

  • Creates, edits, publishes, archives, and deletes prompts
  • Creates and edits prompt categories
  • Filters by status, brand, category, and search

How To Use It

  1. Open Library / Prompts.
  2. Filter the list or add a new prompt.
  3. Edit the content and publish state.
  4. Use the categories area to manage prompt categories separately.

12. Guide Library

What It Is

The brand-scoped guide management area, backed by the article tables.

What It Does

  • Creates, edits, publishes, archives, and deletes guides
  • Creates and edits guide categories
  • Stores slug, summary, content, category, and tags

How To Use It

  1. Open Library / Guides.
  2. Filter by status, brand, category, and search.
  3. Edit the guide content and publish state.

13. Article Library

What It Is

A brand-scoped article management surface that parallels guides but is presented as its own library screen.

What It Does

  • Creates, edits, publishes, archives, and deletes articles
  • Lets operators manage article categories
  • Stores summary, slug, content, and tags for each article

How To Use It

  1. Open Library / Articles.
  2. Filter or search the list.
  3. Select an article or click Add Article.
  4. Edit the summary, content, category, and publish state.

14. Scheduling Series

What It Is

The top-level schedule program screen for recurring or one-off event series.

What It Does

  • Creates event series such as seasonal weekly programs
  • Stores series timezone, season window, recurrence pattern, and day window
  • Controls whether a series is draft or active

How To Use It

  1. Open Scheduling / Series.
  2. Create a series with name, slug, timezone, season dates, and recurrence pattern.
  3. Review generated occurrences and make sure the time window is correct before scheduling participants.

15. Scheduling Resources

What It Is

The management screen for named schedule locations such as stages, food spots, and market booths.

What It Does

  • Creates named resources
  • Classifies resources by type
  • Controls display order and active/inactive state

How To Use It

  1. Open Scheduling / Resources.
  2. Create each stage or booth as a first-class resource.
  3. Keep resource names and slugs stable because planner, calendar, and public feeds use them.

16. Scheduling Participants

What It Is

The schedulable participant projection used by planner and assignments for entertainers, food vendors, and market vendors.

What It Does

  • Represents approved partner accounts as schedulable records
  • Preserves the roster used by planner, calendar, and assignment workflows
  • Separates scheduling projection from partner account/profile management

How To Use It

  1. Use Accounts / Partners and Event Mgmt / Pending Applications to manage participant accounts and approvals.
  2. Use Event Mgmt / Planner or Event Mgmt / Assignments to place the resulting schedulable participants.
  3. The old standalone Event Mgmt / Participants page is retired.

17. Scheduling Assignments

What It Is

The main assignment editor for putting participants onto resources within event occurrences.

What It Does

  • Creates timed entertainment slots
  • Creates full-day vendor placements
  • Stores public-facing titles, subtitles, descriptions, location labels, and URLs outside of the calendar UI
  • Controls draft and published schedule state
  • Supports occurrence-level bulk publish and unpublish

How To Use It

  1. Open Scheduling / Assignments.
  2. Select the occurrence, resource, and participant.
  3. For entertainment, set the exact time slot. For vendors, use the full-day assignment flow.
  4. Fill in the public display fields when the public website should show richer schedule content.
  5. Use bulk publish only after conflicts are resolved.

18. Scheduling Conflicts

What It Is

A dedicated report of overlapping or invalid schedule states.

What It Does

  • Identifies resource overlaps
  • Identifies participant overlaps
  • Supports safe publish-state decisions

How To Use It

  1. Open Scheduling / Conflicts.
  2. Review the reported issues before publishing an occurrence.
  3. Fix assignments in the assignments screen or planner.

19. Scheduling Calendar

What It Is

The first calendar surface for visualizing scheduled assignments over time.

What It Does

  • Displays assignments in calendar form using FullCalendar OSS
  • Supports date and occurrence review
  • Provides an operator-friendly visual cross-check against the assignment table

How To Use It

  1. Open Scheduling / Calendar.
  2. Navigate to the relevant date or occurrence.
  3. Use the calendar to review the schedule visually, then use assignments for deeper edits.

20. Scheduling Planner

What It Is

The resource-first planner view for stages and booths.

What It Does

  • Shows rows for named resources
  • Shows timed entertainment blocks and full-day vendor occupancy
  • Highlights resource utilization and conflicts
  • Provides quick links into assignment management

How To Use It

  1. Open Scheduling / Planner.
  2. Select the occurrence and review the resource rows.
  3. Use this page to answer resource-centric questions such as “what is on this stage today?” or “is this booth already occupied?”.

21. Common Operating Rules

  • Use superadmin accounts for install-wide administration only.
  • Use staff accounts when you want brand-scoped operational access.
  • Do not use Configs to “edit” runtime state; it is intentionally read-only.
  • Treat Brand configuration as live runtime config: host and email mistakes affect real behavior.
  • Use the public-api contract for website integration; do not integrate websites against admin-web internal routes.
  • Resolve schedule conflicts before publishing occurrences.
