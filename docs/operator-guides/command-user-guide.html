1. Getting Started

What It Is

Command is the administrative backoffice for managing brands, operators, public client accounts, content, leads, and installation diagnostics.

What It Does

  • Controls who can access the backoffice
  • Controls brand runtime settings such as hostnames and brand email configuration
  • Stores and manages prompts and guides
  • Shows lead and analytics data

How To Use It

  1. Open the backoffice sign-in page.
  2. Sign in with your backoffice credentials.
  3. If MFA is enabled on your account, complete the authenticator code challenge.
  4. Use the left navigation to move between Dashboard, Analytics, Leads, Accounts, Library, and Settings.
Superadmins can manage all brands and operator accounts. Staff users are automatically limited to the brands assigned to them.

2. Dashboard

What It Is

The operational summary screen for signups, logins, login IP activity, and signup-country breakdowns.

What It Does

  • Shows period-based metrics for Today, 7 Days, and 30 Days
  • Shows top signup countries from stored login geography
  • Shows top login IPs from recorded login events

How To Use It

  1. Open Dashboard.
  2. Select the time period from the period buttons.
  3. Review headline counts, the trend chart, country summary, and login IP table.
  4. Use this page for operational monitoring, not editing.

3. Analytics

What It Is

A read-only lead analytics view built from the lead-event stream.

What It Does

  • Shows deduped total leads
  • Separates contact leads and chat leads
  • Shows a 7-day summary and brand breakdown

How To Use It

  1. Open Analytics.
  2. Click Refresh to reload the current numbers.
  3. Use the top cards for totals and the lower table for brand comparison.

4. Leads

What It Is

A searchable operational table of recent contact and chat leads.

What It Does

  • Filters by source and brand
  • Searches by name, email, and other displayed values
  • Exports the visible result set to CSV or JSON
  • Allows operators to copy the raw JSON payload for a lead row

How To Use It

  1. Open Leads.
  2. Filter by source or brand as needed.
  3. Use search to narrow the table.
  4. Export the current filtered result set when you need to share or archive it.

5. Staff Accounts

What It Is

The operator-management screen for backoffice staff and superadmin identities.

What It Does

  • Creates staff and superadmin accounts
  • Assigns brand access to staff
  • Blocks, unblocks, and deletes accounts
  • Generates invite links and password reset links
  • Resets authenticator MFA when needed

How To Use It

  1. Open Accounts / Staff.
  2. Select an existing user from the directory, or click Add Staff.
  3. Set username, email, role, password, and brand assignments.
  4. Use Create & Invite when you want a one-time invite link instead of setting final credentials manually.
  5. Use Reset MFA only when an operator has lost access to their authenticator setup.

6. Client Accounts

What It Is

The management screen for brand-scoped external/public users.

What It Does

  • Creates and edits external client accounts
  • Assigns them to a brand
  • Marks email as verified when needed
  • Blocks, unblocks, or deletes accounts

How To Use It

  1. Open Accounts / Clients.
  2. Select a client account or click Add Client.
  3. Enter client name, email, brand, and password if creating a new account.
  4. Use Mark Verified only when you intentionally need to bypass the normal email-verification flow.
  5. Use Block or Unblock to control sign-in eligibility.

7. Brands

What It Is

The live source-of-truth screen for brand identity, runtime host mapping, and brand email configuration.

What It Does

  • Creates and edits brands
  • Stores production and preview public/admin hosts
  • Stores apex host routing
  • Stores brand email settings such as status, from name/email, reply-to, support email, and provider secret reference

How To Use It

  1. Open Settings / Brands.
  2. Select a brand or click Add Brand.
  3. Set the brand key and brand name.
  4. Configure production and preview public/admin hosts.
  5. Set brand email configuration and activate it only when email is ready for that brand.
  6. Save the brand. Runtime host recognition depends on these saved database values.

8. Configs

What It Is

A read-only diagnostics screen for install-level runtime and service configuration.

What It Does

  • Shows env presence or absence
  • Shows masked/fingerprinted secrets and database URL summaries
  • Shows runtime status and database connectivity checks
  • Shows what the running installation is actually using

How To Use It

  1. Open Settings / Configs.
  2. Review the runtime, auth, and service groups.
  3. Use this screen to confirm state, not to edit values.
  4. Go to Settings / Brands for brand-level editable runtime config.

9. Security

What It Is

The per-operator MFA management screen for authenticator-app enrollment.

What It Does

  • Shows MFA status, method, enablement date, and recovery-code generation state
  • Starts MFA setup
  • Displays QR code, setup secret, setup URI, and recovery codes during enrollment
  • Verifies setup with a 6-digit authenticator code

How To Use It

  1. Open Settings / Security.
  2. If MFA is disabled, click Set Up Authenticator.
  3. Scan the QR code or copy the setup secret into your authenticator app.
  4. Save the recovery codes before completing setup.
  5. Enter a current 6-digit code to finish enrollment.

10. Prompt Library

What It Is

The brand-scoped prompt management area.

What It Does

  • Creates, edits, publishes, archives, and deletes prompts
  • Creates and edits prompt categories
  • Filters by status, brand, category, and search
  • Keeps category uniqueness within each brand only

How To Use It

  1. Open Library / Prompts.
  2. Use the top filters to narrow the list.
  3. Select an existing prompt or click Add Prompt.
  4. Set title, status, brand, category, description, and content.
  5. Save when ready. Use the Categories card to maintain prompt categories separately.

11. Guide Library

What It Is

The brand-scoped guide management area, backed by the article tables.

What It Does

  • Creates, edits, publishes, archives, and deletes guides
  • Creates and edits guide categories
  • Stores slug, summary, content, category, and tags
  • Enforces brand-scoped uniqueness for category names, category slugs, and guide slugs

How To Use It

  1. Open Library / Guides.
  2. Filter by status, brand, category, and search.
  3. Select a guide or click Add Guide.
  4. Set title, slug, status, brand, guide category, tags, summary, and content.
  5. Use the lower Guide Categories section to manage guide categories independently.

12. Common Operating Rules

  • Use superadmin accounts for install-wide administration only.
  • Use staff accounts when you want brand-scoped operational access.
  • Do not use Configs to “edit” runtime state; it is intentionally read-only.
  • Treat Brand configuration as live runtime config: host and email mistakes there affect real behavior.
  • Use the public-api contract for website integration; do not integrate websites against admin-web internal routes.
